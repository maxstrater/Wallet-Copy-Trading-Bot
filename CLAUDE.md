# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Polymarket wallet copy-trading bot — monitors target wallets on Polymarket and mirrors their trades automatically.

## Commands

```bash
# Install dependencies
pip install -r requirements.txt

# Run the bot
python main.py

# Run with a specific config
ENV_FILE=.env.prod python main.py
```

## Architecture

- `main.py` — entry point, main loop that polls for new trades and dispatches copy orders
- `config.py` — loads and validates all environment variables (API keys, wallet addresses, trade sizing)
- `db.py` — persistence layer for tracking seen trades and positions, preventing duplicate execution
- All secrets are loaded exclusively through `config.py` from a `.env` file — never hardcoded

## Environment

Copy `.env.example` to `.env` and fill in values before running.

## Prompt 2:
"Implement wallet_monitor.py for the Polymarket copy-trading bot.

This module watches a list of target wallet addresses and detects new
trades in real time by polling the Polymarket Data API. No external AI
APIs are used — this is pure data fetching and filtering.

DATA SOURCES:
- Primary: Polymarket Data API
  GET https://data-api.polymarket.com/activity?user={wallet_address}&limit=20
  Returns recent on-chain activity for a wallet address.
- Market metadata: Gamma Markets API
  GET https://gamma-api.polymarket.com/markets?id={condition_id}
  Returns market details including question, category, resolution status,
  end date, and liquidity.
- Do NOT use the CLOB API for monitoring — it is for order placement only.

WALLETS CONFIG:
Load target wallets from wallets.json in this exact format:
  {
    "wallets": [
      {
        "address": "0x...",
        "label": "whale_1",
        "copy_ratio_override": null
      }
    ]
  }
- If wallets.json is missing or has an empty wallets array, log a warning
  and return an empty list. Do not crash.
- Reload wallets.json on every poll cycle so new wallets can be added
  without restarting the bot.

TRADE DETECTION LOGIC:
On each poll cycle:
1. For each wallet, fetch its last 20 activities from the Data API
2. Track the most recently seen trade timestamp per wallet in a dict
   keyed by wallet address (persists in memory across poll cycles)
3. Only process activities NEWER than the last seen timestamp
4. A valid tradeable activity must satisfy ALL of:
   - type field is "trade" (not "redeem", "transfer", or "merge")
   - outcome field is "YES" or "NO"
   - usdcSize >= 10.0
   - The market is NOT yet resolved (resolved == false from Gamma API)
   - Market end date is MORE than 3 hours from now
   - Market end date is LESS than 120 days from now (avoid illiquid long-shots)
   - Market liquidity from Gamma API >= 500 USDC (avoids thin markets)

## Prompt 3:
"Implement wallet_scorer.py for the Polymarket copy-trading bot.

This module fetches historical trade data for target wallets, calculates
quantitative performance metrics, and stores them in the database.
No external AI APIs are used — all calculations are pure Python and numpy.

DATA SOURCES:
- Polymarket Data API (paginated):
  GET https://data-api.polymarket.com/activity?user={address}&limit=100&offset={n}
  Paginate until you have up to 500 trades or the response returns fewer
  than 100 results (meaning you've reached the end).
- Gamma API for market resolution data:
  GET https://gamma-api.polymarket.com/markets?id={condition_id}
  Use this to determine if a traded market resolved YES or NO.

METRICS TO CALCULATE (over the last 90 days of trades):

1. win_rate (float 0.0–1.0):
   Count resolved trades where the wallet bet on the correct outcome.
   win_rate = winning_resolved_trades / total_resolved_trades
   Require at least 10 resolved trades — return None if fewer exist.

2. total_bets (int):
   Total number of trade activities in the last 90 days (open + resolved).

3. avg_roi (float):
   Mean return on investment across all resolved trades.
   Winning trade ROI = (1.0 - entry_price) / entry_price
   Losing trade ROI  = -1.0
   Return the arithmetic mean.

4. consistency_score (float 0.0–1.0):
   Split the 90-day window into three 30-day buckets.
   Calculate win_rate per bucket (skip buckets with fewer than 3 resolved trades).
   consistency_score = 1.0 - numpy.std(bucket_win_rates)
   A wallet that wins 60% every month scores higher than one that wins
   90% one month and 30% the next.

5. avg_bet_size (float):
   Mean usdcSize across all trades in the 90-day window.
   Useful for normalising copy sizes.

6. market_categories (str):
   Top 3 categories by trade count, comma-separated.
   e.g. "crypto,politics,sports"

7. hot_streak (int):
   Count the number of consecutive winning resolved trades at the END of the
   wallet's history (most recent first). Reset to 0 at the first loss.
   A wallet on a 5-trade win streak is a stronger signal than one that was
   good 2 months ago.

8. recency_weight (float 0.0–1.0):
   Fraction of the wallet's total bets that occurred in the last 30 days
   vs the full 90-day window. Higher = more recently active.
   recency_weight = bets_last_30_days / max(total_bets, 1)

COMPOSITE SCORE (float 0.0–1.0):
Combine the above metrics into a single overall score:
  score = (
    win_rate           * 0.35 +
    consistency_score  * 0.25 +
    roi_factor         * 0.20 +
    recency_weight     * 0.10 +
    streak_factor      * 0.10
  )
  where:
    roi_factor    = clamp(avg_roi / 0.6, 0.0, 1.0)
    streak_factor = clamp(hot_streak / 5.0, 0.0, 1.0)

CACHING:
- Store all metrics in the wallet_scores table via db.upsert_wallet_score()
- Only re-fetch from the API if last_updated is more than 6 hours ago
- On startup, refresh all wallets. During runtime, refresh every 6 hours
  via the scheduler in main.py.

WalletScore DATACLASS:
  wallet_address: str
  win_rate: float
  total_bets: int
  avg_roi: float
  consistency_score: float
  avg_bet_size: float
  market_categories: str
  hot_streak: int
  recency_weight: float
  composite_score: float
  last_updated: datetime

EXPOSED INTERFACE:
WalletScorer class with:
  __init__(self, config: Config, db: Database)
  score_wallet(self, wallet_address: str) -> WalletScore | None
    Fetches data, calculates all metrics, stores in DB, returns WalletScore.
    Returns None if there is insufficient resolved trade data.
  refresh_all(self, wallet_addresses: List[str]) -> None
    Scores all wallets. Skips wallets whose score is fresh (< 6 hours old).
    Logs a summary table at the end showing each wallet's composite score.
  get_score(self, wallet_address: str) -> WalletScore | None
    Returns cached score from DB without hitting the API.
    Returns None if the wallet has never been scored.

BEST PRACTICES:
- Never crash on a single wallet failure — catch all exceptions, log at
  WARNING, and continue to the next wallet
- Sleep 0.5 seconds between API calls to avoid rate limiting
- If the Gamma API returns no resolution data for a market, exclude that
  trade from win_rate and roi calculations (don't count it as a loss)
- Log each wallet's final composite score at INFO level

Use Config from config.py, Database from db.py,
numpy for std calculations, structlog from utils.py."

## Prompt 4:
"Implement signal_engine.py for the Polymarket copy-trading bot.

This is the quantitative core of the bot. It takes a detected trade and a
wallet score and computes a set of named numeric signals that collectively
determine how strong the copy opportunity is. Each signal is a float from
0.0 to 1.0. The signals are weighted and summed into a final score 0–100.

This module has NO external API calls and NO AI dependencies.
All logic is deterministic, fast, and fully explainable.

INPUT:
  trade: NewTrade         (from wallet_monitor.py)
  wallet_score: WalletScore   (from wallet_scorer.py)
  open_positions: list    (from db.get_open_positions())
  available_usdc: float   (current balance)

SIGNALS (compute all of these):

Signal 1 — wallet_quality (weight: 25)
  Value = wallet_score.composite_score
  Rationale: directly reflects historical performance.

Signal 2 — price_efficiency (weight: 20)
  Value = 1.0 - abs(trade.price - 0.5) * 2
  Range: 0.0 (price at 0.05 or 0.95) to 1.0 (price at exactly 0.50)
  Rationale: markets priced near 0.5 are genuinely uncertain and have the
  most remaining value. Prices near the extremes are nearly resolved.

Signal 3 — bet_size_conviction (weight: 15)
  Value = clamp(trade.size_usdc / (wallet_score.avg_bet_size * 2), 0.0, 1.0)
  Rationale: if the wallet bet 2x or more their average size, that signals
  unusually high conviction. A bet at their average size scores 0.5.

Signal 4 — time_value (weight: 15)
  hours_remaining = (trade.closes_at - datetime.utcnow()).total_seconds() / 3600
  Value = clamp((hours_remaining - 4) / (72 - 4), 0.0, 1.0)
  Rationale: markets closing in < 4h score 0, closing in 72h+ score 1.
  Sweet spot is 12–48 hours: enough time for the position to work but
  not so long that liquidity is thin.

Signal 5 — liquidity_depth (weight: 10)
  Value = clamp(trade.liquidity_usdc / 10000, 0.0, 1.0)
  Rationale: markets with $10k+ liquidity are easy to enter and exit.
  Thin markets below $1000 are risky to copy into.

Signal 6 — hot_streak (weight: 10)
  Value = clamp(wallet_score.hot_streak / 5.0, 0.0, 1.0)
  Rationale: a wallet on a winning streak right now is more relevant than
  one that was good last month but cold recently.

Signal 7 — portfolio_fit (weight: 5)
  existing_exposure_this_market = sum of open positions in this market_id
  Value = 1.0 if existing_exposure_this_market == 0 else 0.0
  Rationale: binary — we strongly prefer not to double up on a market.

FINAL SCORE CALCULATION:
  raw_score = sum(signal_value * signal_weight for each signal)
  total_weight = sum of all weights = 100
  final_score = round(raw_score)  # integer 0–100

SIGNAL BREAKDOWN OUTPUT:
Return a SignalResult dataclass containing:
  final_score: int               # 0–100
  signals: List[SignalDetail]    # one per signal, for logging and DB storage

SignalDetail dataclass:
  name: str                      # e.g. "wallet_quality"
  value: float                   # 0.0–1.0
  weight: int                    # e.g. 25
  contribution: float            # value * weight, e.g. 18.5
  description: str               # one human-readable sentence

HUMAN-READABLE SUMMARY:
Include a method generate_reasoning(signal_result: SignalResult) -> str
that produces a single paragraph explaining the score in plain English.
It should identify the top 2 strongest signals and the weakest signal.
Example output:
  "Score 74/100. Strong signals: wallet quality (18.5/25) — 67% win rate
   with consistent performance; conviction (11.2/15) — bet 2.3x their
   average size. Weakest: hot streak (2.0/10) — wallet is not currently
   on a winning run."
This string is used in Telegram alerts so you understand each decision.

EXPOSED INTERFACE:
SignalEngine class with:
  __init__(self, config: Config)
  compute(self, trade, wallet_score, open_positions, available_usdc) -> SignalResult
    Computes all 7 signals, calculates final score, generates reasoning.
    Logs the full signal breakdown at DEBUG level.
    Never raises — if a signal calculation fails, set it to 0.0 and log
    a WARNING with the signal name and error.

Write unit tests in test_signal_engine.py that verify:
  1. A trade with price=0.50 scores higher on price_efficiency than price=0.80
  2. A bet at 3x average size scores higher on conviction than 1x average
  3. A market closing in 2 hours scores lower on time_value than 48 hours
  4. An existing position in the same market causes portfolio_fit = 0
  5. A wallet on a 5-trade hot streak scores 1.0 on hot_streak signal
  6. Total weights always sum to 100

Run the tests immediately after writing them and fix any failures.
Use Config from config.py and structlog from utils.py."

## Prompt 5:
"Implement decision_engine.py for the Polymarket copy-trading bot.

This module combines the wallet score and signal engine output into a final
copy/skip decision with position sizing. It is pure Python logic — no
external APIs, no AI, no network calls of any kind.

INPUT:
  trade: NewTrade
  wallet_score: WalletScore | None
  signal_result: SignalResult       (from signal_engine.py)
  open_positions: list              (from db.get_open_positions())
  available_usdc: float

HARD GATE RULES (apply in order — first failure = skip immediately):

GATE 1 — Wallet data available:
  Skip if wallet_score is None.
  skip_reason = "no_wallet_data"

GATE 2 — Wallet minimum quality:
  Skip if wallet_score.win_rate < config.MIN_WALLET_WIN_RATE
  Skip if wallet_score.total_bets < config.MIN_WALLET_BETS
  skip_reason = "wallet_below_threshold"

GATE 3 — Minimum signal score:
  Skip if signal_result.final_score < config.MIN_SIGNAL_SCORE
  skip_reason = "signal_score_too_low"

GATE 4 — Market timing:
  Skip if market closes in less than 3 hours
  Skip if market closes in more than 90 days
  skip_reason = "market_timing"

GATE 5 — Price sanity:
  Skip if trade.price < 0.04
  Skip if trade.price > 0.96
  skip_reason = "price_out_of_range"

GATE 6 — Duplicate position:
  Skip if there is already an open position in this market_id AND side
  skip_reason = "duplicate_position"

GATE 7 — Portfolio exposure cap:
  total_exposure = sum of open positions' size_usdc
  Skip if total_exposure >= config.MAX_PORTFOLIO_EXPOSURE_USDC
  skip_reason = "max_exposure_reached"

GATE 8 — Minimum capital:
  Skip if available_usdc < 25.0
  skip_reason = "insufficient_capital"

POSITION SIZING (only reached if all gates pass):
  base_size   = trade.size_usdc * config.COPY_RATIO
  cap_by_max  = config.MAX_POSITION_SIZE_USDC
  cap_by_pct  = available_usdc * 0.12        # never more than 12% of balance
  cap_by_room = config.MAX_PORTFOLIO_EXPOSURE_USDC - total_exposure
  final_size  = round(min(base_size, cap_by_max, cap_by_pct, cap_by_room), 2)

  If final_size < 10.0:
    skip with reason = "position_too_small"

SIGNAL SCORE TIERING:
  Map the signal score to a confidence label for display:
    >= 85: "very high"
    >= 70: "high"
    >= 55: "medium"  (MIN_SIGNAL_SCORE should be set above this in practice)
    <  55: "low"

Decision DATACLASS:
  action: str                    # "copy" or "skip"
  side: str | None               # "YES" or "NO" (None if skip)
  size_usdc: float | None        # (None if skip)
  signal_score: int              # always present (0–100)
  confidence_label: str          # "very high" / "high" / "medium" / "low"
  skip_reason: str | None        # populated if skip
  reasoning: str                 # from signal_engine.generate_reasoning()
  trade: NewTrade
  wallet_score: WalletScore | None
  signal_result: SignalResult

EXPOSED INTERFACE:
DecisionEngine class with:
  __init__(self, config: Config, db: Database, signal_engine: SignalEngine)
  evaluate(self, trade, wallet_score, available_usdc) -> Decision
    Fetches open positions from DB internally.
    Runs all 8 gates in order.
    If all pass, calculates position size.
    Returns a Decision in all cases.

LOGGING:
Log every decision at INFO level:
  action=copy  wallet={label} score={signal_score} size=${size} market="{question[:55]}"
  action=skip  wallet={label} score={signal_score} reason={skip_reason} market="{question[:55]}"

Write unit tests in test_decision_engine.py that cover:
  1. wallet_score=None → skip with "no_wallet_data"
  2. win_rate below threshold → skip with "wallet_below_threshold"
  3. signal_score below MIN_SIGNAL_SCORE → skip with "signal_score_too_low"
  4. duplicate position same market+side → skip with "duplicate_position"
  5. available_usdc=15 → skip with "insufficient_capital"
  6. portfolio at max exposure → skip with "max_exposure_reached"
  7. valid trade → action="copy", size_usdc respects all three caps
  8. final_size rounds to 2 decimal places
  9. confidence_label maps correctly at 85, 70, 55 boundaries

Run the tests immediately and fix any failures before finishing.
